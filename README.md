# OpenToonz
**Fast Brief:** Professional 2D animation suite — the same tool behind Studio Ghibli films, now free and open source.

## Overview

OpenToonz is the open-source release of Toonz, the animation production software used by Studio Ghibli, Rough Draft Studios, and countless independent animators. It combines traditional frame-by-frame hand-drawn animation with modern digital compositing, effects, and a node-based workflow engine that rivals proprietary packages.

The application ships with GTS, a dedicated scanning tool for digitizing hand-drawn paper animation, plus raster and vector drawing engines within a unified timeline. Its FX schematic editor connects effects like blur, glow, and color correction through a visual node graph, while the plastic tool enables cutout-style bone-based rigging for character animation.

## Getting Started

Download OpenToonz from the official GitHub releases page. Launch the application and create a new scene from the startup dialog. Use the default room layout or customize panels via Windows > Workspace. Start a raster level, select a brush from the tool palette, and draw directly on the canvas. Turn the onion skin on to see adjacent frames for smooth in-betweening.

## Everyday Use

1. Create a new scene with your chosen resolution and frame rate.
2. Add a new raster or vector level via the Xsheet right-click menu.
3. Draw key poses on frames 1, 5, 9, etc., using the brush and fill tools.
4. Enable onion skinning to draw in-between frames with visual reference.
5. Use the FX Schematic to add shadows, glows, and color grading filters.
6. Render the animation to a video file via Render > Preview or Render > Output Settings.

## Scenarios

**A. Hand-Drawn Character Animation:** Scan paper drawings via GTS or draw directly in a raster level, block out key poses on the Xsheet, use onion skin to draw breakdowns, ink and paint with the fill tool, and composite backgrounds in the schematic for a polished short film.

**B. Cutout-Rigged Character:** Import a character broken into body-part layers, use the Plastic tool to define a skeleton hierarchy, set keyframes for limb rotation, auto-tween motion between poses, and export a looping character animation for a game sprite asset.

**C. Multi-Planer Camera Shot:** Place foreground, midground, and background layers in separate columns of the Xsheet, assign each column a different Z-depth, use the camera tool to dolly through the scene, and apply depth-of-field blur via the FX schematic for a cinematic establishing shot.

**D. Particle-Based Effects:** Create a particle level for rain or snow, modify the particle emitter properties to control density, velocity, and lifespan, attach a wind force modifier, and render the environmental effect layer for compositing over a static painted background.

## Troubleshooting

1. **Drawing lag with large brush sizes:** Reduce the brush size temporarily during rough sketching; switch from raster to vector levels for responsive line art, or lower the canvas preview resolution in Preferences.
2. **Onion skin does not show adjacent frames:** Verify that onion skin is toggled on in the viewer toolbar and that the keyframes are populated with drawings; adjust the onion skin range using the slider at the bottom of the viewer.
3. **Render output is blank:** Check that the output folder is writable, the correct columns are marked for rendering in the Xsheet, and the render range covers frames that actually contain drawn content.
4. **FX schematic nodes don't update in the viewer:** Confirm that the schematic output node is connected to the viewer column; click the camera icon on the schematic toolbar to force a preview re-render.
5. **GTS scanner is not detected:** Ensure the scanner TWAIN or WIA driver is installed on the system; GTS works with most flatbed scanners — test the scanner in another application first to isolate driver issues.
